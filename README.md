## 一、七虾是什么？

**七虾**是一套基于 OpenClaw AI 平台的**自媒体内容自动生产流水线**。它不是单一的 AI 提示词集合，而是一套**完整的、自动化的内容生产系统**，覆盖从选题构思到最终审核的全流程。

### 七虾能做什么？

| 能力 | 说明 |
|:----|:-----|
| **自动产出** | 输入一个主题，自动完成选题→框架→大纲→正文→标题→配图→审核 |
| **多平台适配** | 同时产出公众号、小红书、头条号、知乎适配版本 |
| **质量可控** | 手动模式下每步确认，审核不通过自动精准回退 |
| **中断恢复** | 跑一半断开？状态文件自动保存，一键恢复 |

### 核心理念

**"灵感是耗材，系统是资产。"**

传统内容创作靠"灵感来了写一篇"，七虾把创作变成了**系统化生产**——你只需要给出主题方向，剩下的交给流水线。

## 二、整体架构与流水线全景

### 管线全景

```
用户需求 → ①选题虾 → ②框架虾 → ③大纲虾 → ④文案虾 → ⑤标题虾 → ⑥图片虾 → ⑦审核虾 → 闭环
                                                                                      │
                                                                                审核不通过
                                                                                 ↓ 精准回退
```

### 七虾职责速览

| 环节 | 名称 | 职责 | 核心产出 |
|:----:|:----|:-----|:--------|
| ① | 选题虾 | 挖掘主题、分析热点、整理口水稿 | 选题清单 |
| ② | 框架虾 | 按选题推荐最适合的写作框架 | 选定框架+说明 |
| ③ | 大纲虾 | 按框架生成文章骨架（含金句位置） | 完整大纲 |
| ④ | 文案虾 | 写正文、去 AI 味润色、多平台改写 | 完整稿件 |
| ⑤ | 标题虾 | 各平台标题创作（每平台3条） | 多平台标题方案 |
| ⑥ | 图片虾 | 封面图+配图提示词生成 | 图片提示词方案 |
| ⑦ | 审核虾 | 十维度终审，不通过则精准回退 | 审核报告 |

### 五个写作框架

| 框架 | 全称 | 适用场景 | 结构 |
|:----|:-----|:---------|:-----|
| **P&P** | Point & Proof / Problem & Process | 观点输出、经验分享、教程 | 观点+证据 / 问题+过程 |
| **PAS** | Pain → Agitate → Solution | 营销干货、引流、共鸣文 | 痛点→激化→解决 |
| **PPP** | Promise → Proof → Proposal | 深度长文、教程、视频脚本 | 承诺→证明→提议 |
| **故事框架** | — | 叙事、案例、品牌故事 | 开端→发展→转折→结局 |
| **清单框架** | — | 盘点、排名、合集类 | 总起→逐条展开→总结 |

#### 各框架详解

**P&P（观点+证据 / 问题+过程）**
```
Point（观点）：一句话亮出核心论点
    ↓
Proof（证据）：用案例、数据、故事支撑
    ↓
Practical（实操）：给读者即刻能做的行动
```

适合：**观点输出、行业分析、书评/影评、日常思考**
示例："靠灵感做内容注定失败。Dan Koe 做了200多个视频后发现，有系统的创作者和不系统的产出量差7倍。今天你就能用选题矩阵替换掉'今天写什么'的焦虑。"

---

**PAS（痛点—激化—解决）**
```
Pain（痛点）：精准描述读者的痛苦
    ↓
Agitate（激化）：放大痛点，让读者感同身受
    ↓
Solution（解决）：给出你的方法/观点/产品
```

适合：**引流内容、转化文案、爆款开头、共鸣型内容**
示例：你花了3小时写的内容，发出去只有50个阅读。更糟的是，你明明看到别人的内容和你差不多，却爆了。你开始怀疑自己……问题不在你，在你没有内容系统。

---

**PPP（承诺—证明—提议）**
```
Promise（承诺）：告诉读者读完能获得什么
    ↓
Proof（证明）：用逻辑/案例/数据证明承诺可信
    ↓
Proposal（提议）：给出具体、可执行的步骤
```

适合：**深度长文、视频脚本、课程大纲**
各部分占比：承诺5-10%，证明20-30%，提议60-70%

---

### 写作框架快速选择

| 你要写什么 | 首选框架 | 备选框架 |
|:----------|:--------|:---------|
| 一条引流推文 | PAS | P&P |
| 一篇深度长文 | PPP | P&P + PAS |
| 视频脚本 | PPP（整体）+ PAS（开头） | — |
| 日常思考分享 | P&P | — |
| 经验复盘/教程 | Problem & Process | PPP |
| 产品介绍/转化页 | PAS | PPP |

## 三、各虾技能详解

### ① 选题虾

**核心能力**：帮你解决"今天写什么"的问题。

**工作流程**：
1. 用户输入主题方向（或由选题虾自动分析）
2. 选题虾分析并产出选题清单
3. 如果开启"口水稿"模式，还会产出初步的口语化素材

**输入**：主题关键词、平台类型、写作目标
**产出**：选题清单（含选题方向、切入点、受众分析）

**使用时机**：
- 不知道写什么的时候
- 想追热点但不知道怎么切入
- 需要系统规划未来1-2周的内容选题

**调用方式**：
```
帮我看看最近有什么好话题（适合XX平台）
我要写一篇关于XX的文章，帮我找几个选题方向
```

---

### ② 框架虾

**核心能力**：为你的选题匹配最适合的写作框架。

**工作流程**：
1. 接收选题虾产出的选题方向
2. 分析选题类型（教程/观点/营销/故事/盘点）
3. 从5大框架中推荐最佳框架
4. 给出框架选择理由和结构说明

**输入**：选题方向、文章类型、平台
**产出**：选定框架 + 框架结构说明 + 使用建议

**核心原则**：不同类型的选题匹配不同的框架，框架选对了，文章成功了一半。

**调用方式**：
```
这个选题适合什么框架？
帮我选一个框架：主题是XX，平台是公众号
```

---

### ③ 大纲虾

**核心能力**：把框架变成可执行的文章骨架。

**工作流程**：
1. 接收框架虾的框架选择
2. 按框架结构生成完整大纲
3. 标注每个部分的写作要点和字数建议
4. 标注金句/案例/数据插入位置
5. **产出大纲后等待用户确认**，确认后才进入正文写作

**输入**：框架 + 选题 + 平台
**产出**：完整大纲（含各部分要点、字数配比、金句位置）

**⚠️ 关键规则**：大纲必须由用户确认后方可进入下一步。不满意可反复修改。

**调用方式**：
```
帮我搭个大纲
为这个选题写一份大纲：主题XX，框架PPP
```

---

### ④ 文案虾

**核心能力**：把大纲变成真正能发出去的稿件，并适配多平台。

**工作流程**：
1. 接收确认后的大纲
2. 按大纲逐部分展开写作
3. 去 AI 味润色（消除"首先/其次/最后""值得注意的是"等套话）
4. 根据目标平台特点进行改写适配
5. 多平台需求：产出公众号版、小红书版、头条号版

**五大写作原则**：
- ✅ 口语化：像朋友聊天一样自然
- ✅ 短段落：每段2-4行，移动端友好
- ✅ 有钩子：每200-300字一个情绪钩子
- ✅ 配案例：道理+故事结合
- ✅ 去AI味：删除套话，加入个人风格

**输入**：确认的大纲 + 框架 + 平台
**产出**：完整稿件（多版本）

**调用方式**：
```
帮我写正文，基于这份大纲
根据大纲写一篇公众号文章
把这篇改写一下，适配小红书
```

---

### ⑤ 标题虾

**核心能力**：为文章创作多平台爆款标题。

**工作流程**：
1. 基于正文内容分析核心卖点
2. 为每个目标平台创作3条标题
3. 自动选定最终标题

**平台标题策略**：

| 平台 | 标题核心 | 公式 | 长度 |
|:----|:--------|:-----|:---:|
| 头条号 | 制造好奇心/情绪钩子 | 悬念型/冲突型/情绪型 | 前15字抓人 |
| 小红书 | 精准命中搜索词 | 痛点型/数字型/互动提问型 | 20字以内 |
| 公众号 | 搜索词+痛点+承诺 | 关键词前置+价值承诺 | 适中 |
| 知乎 | 选对问题比写好标题重要 | 问题本身包含热词 | — |

**六种爆款标题公式**：

| 公式类型 | 格式 | 示例 |
|:--------|:----|:------|
| **悬念型** | 否定词+反常+结果承诺 | "90%人不知道的微信隐藏功能" |
| **痛点型** | 提问+解决方案 | "Excel 做工资条太慢？5分钟搞定" |
| **数字型** | 数字+干货承诺 | "这5个DeepSeek技巧让效率翻3倍" |
| **冲突型** | 反差+利益点 | "月薪3万的程序员深夜痛哭" |
| **结果型** | 结果展示+场景 | "我用这个方法3天涨粉5000" |
| **情绪型** | 共情+解决方案 | "做了3个月小红书，终于搞懂了SEO" |

**输入**：正文稿件 + 目标平台
**产出**：多平台标题方案（每平台3条）

**调用方式**：
```
拟几个标题
为这篇文章写标题：公众号+小红书
```

---

### ⑥ 图片虾

**核心能力**：为文章生成中英双语的配图提示词。

**工作流程**：
1. 基于文章内容理解核心视觉元素
2. 根据目标平台生成封面图提示词
3. 生成文中配图提示词
4. 提示词格式：中英双语 + 风格统一 + 尺寸适配
5. 在正文中插入图片占位符 `**【MEDIA:...】**`

**各平台封面尺寸**：

| 平台 | 封面比例 | 封面推荐尺寸 | 说明 |
|:----|:--------:|:-----------:|:-----|
| 公众号 | **2.35:1** | 900×383px | 信息流裁切中央方形区域 |
| 头条号 | **16:9** | 1200×675px | 横版 |
| 小红书 | **3:4** | 1080×1440px | 竖版优先 |
| 知乎 | **16:9** | 1920×1080px | 关键信息居中 |
| 知识星球 | **1:1** | 1080×1080px | 正方形 |

**⚠️ 封面安全区**：
- 公众号：历史消息/朋友圈分享时自动裁切成中央方形，重要内容必须居中
- 知乎：信息流中显示中央区域，关键信息居中布局

**配图提示词结构**：
```
英文提示词（200-300字符）：详细描述画面、构图、光影、色彩
中文提示词：凝练版，清晰传达画面内容
```

**输入**：正文稿件 + 平台 + 风格偏好
**产出**：图片提示词方案（封面+配图）+ 正文占位符

**调用方式**：
```
生成封面提示词
给这篇文章配图生成提示词，风格简约现代风
```

### ⑦ 审核虾

**核心能力**：十维度全面终审，不通过精准回退。

**工作流程**：
1. 接收整篇文章（标题+正文+配图方案）
2. 按十维度逐项审核评分
3. 不通过则精准回退到对应环节

**十维度审核体系**：

| # | 维度 | 优先级 | 说明 |
|:-:|:----|:-----:|:-----|
| ① | **标题吸引力** | 🔴 P0 | 是否让人想点进去 |
| ② | **小标题与结构逻辑** | 🟡 P1 | 各部分之间是否逻辑递进 |
| ③ | **数据准确性** | 🔴 P0 | 数据是否有来源、有时效 |
| ④ | **标题-稿件一致性** | 🟡 P1 | 标题承诺是否在正文兑现 |
| ⑤ | **图片提示词质量** | 🟡 P1 | 提示词是否清晰描述画面 |
| ⑥ | **标题-图片一致性** | 🟡 P1 | 封面图视觉是否与标题呼应 |
| ⑦ | **表达风格一致性** | 🟡 P1 | 全文风格是否统一 |
| ⑧ | **废话与AI痕迹清除** | 🟡 P1 | 是否删除了AI套话 |
| ⑨ 🔥 | **CMO内容营销优化** | 🟡 P1 | 适配平台推荐算法 |
| ⑩ 🔥 | **SEO搜索引擎优化** | 🟢 P2 | 优化搜索排名 |

**通过条件**：
- P0 问题 = 0（无致命错误）
- P1 问题 ≤ 2（重要问题不超过2个）
- 未触碰内容安全红线

**精准回退规则**：

| 问题类型 | 回退至 |
|:---------|:-------|
| 选题角度/方向 | 选题虾 |
| 写作框架不当 | 框架虾 |
| 大纲问题/方向跑偏 | 大纲虾 |
| 稿件质量/结构/AI味 | 文案虾 |
| 标题问题 | 标题虾 |
| 图片提示词问题 | 图片虾 |

**输出**：审核报告（含评分 + 问题清单 + 回退建议）

**调用方式**：
```
帮我审稿
审核一下这篇文章，看能不能发
```

## 四、两种运行模式

### 全自动流水线

一句话触发，编排器自动依次调用七虾，每步自动推进。

**启动指令**：
```
七虾全流程，帮我写一篇关于XX的文章
七虾全流程，公众号，写一篇XX主题的文章
启动七虾，写一篇XX主题的文章，发公众号和小红书
```

**运行特点**：
- 自动依次调用选题→框架→大纲→正文→标题→配图→审核
- 中间无需用户干预
- 全流程自动产出所有文件

### 手动分步模式

每环节完成后**等待用户确认**，确认后再启动下一步。

**启动指令**：
```
七虾手动，写一篇XX主题的文章
手动模式，公众号，XX主题
```

**运行特点**：
- 每完成一个环节停下来等你确认
- 不满意可以要求重做当前环节
- 可随时修改方向，灵活性高
- **铁律：民哥没说"继续""确认""下一步"之前，不得自行推进**

**手动分步流程**：
```
①选题虾完成 → ⏸️ 等确认 → ②框架虾完成 → ⏸️ 等确认
→ ③大纲虾完成 → ⏸️ 需确认大纲（必须确认） → ④文案虾完成 → ⏸️ 等确认
→ ⑤标题虾完成 → ⏸️ 等确认 → ⑥图片虾完成 → ⏸️ 等确认
→ ⑦审核虾完成 → ✅ 发布
```

**全流程各阶段产出文件**：
- 阶段1（选题虾）→ 选题清单
- 阶段2（框架虾）→ 选定框架+说明
- 阶段3（大纲虾）→ 完整大纲
- 阶段4（文案虾）→ 完整稿件
- 阶段5（标题虾）→ 标题方案
- 阶段6（图片虾）→ 图片提示词方案
- 阶段7（审核虾）→ 审核报告

**所有产出文件自动归档到文章文件夹**。

## 五、安装与部署

### 前置需求

- **OpenClaw**：已安装并正常运行的 OpenClaw 平台（版本2026.5.27+）
- **Node.js**：v18+
- **操作系统**：Windows 10/11（macOS/Linux 也可用，安装方式略有不同）

### 安装方式

#### 方式一：一键安装（推荐）

1. 解压 `七虾技能包v3.2.3.zip`
2. 右键 `一键安装脚本.ps1` → **使用 PowerShell 运行**
3. 重启 OpenClaw

#### 方式二：手动安装

1. 解压技能包
2. 将 `skills/` 下的8个文件夹复制到 `~/.openclaw/skills/`
3. 将 `外部依赖/` 下的文件复制到相应的依赖目录
4. 重启 OpenClaw

#### 验证安装

安装后在 OpenClaw 中运行技能检查命令：
```
openclaw skills check
```

或直接向 AI 助手发送：
```
七虾有哪些技能？
```

正常响应说明安装成功。

## 七、注意事项

### 1. 大纲必须确认

大纲虾产出大纲后，**必须由用户确认大纲**，否则不进入正文写作。这是确保内容方向不跑偏的关键节点。

### 2. 手动模式不可自行推进

手动模式下，每完成一个环节后**必须停在原地等待确认**。民哥没说"继续""确认""下一步"之前，不得自行推进到下一环节。

### 3. 封面图安全区

- **公众号封面**：历史消息/朋友圈分享自动裁切成中央方形，重要内容必须居中
- **知乎封面**：信息流显示中央区域，关键信息居中

### 4. 关键词密度控制

- 公众号/知乎：关键词密度控制在 **2%-5%**
- 小红书：关键词密度控制在 **2%-8%**
- 避免堆砌关键词，否则可能限流

### 5. 审核不通过的处理

审核虾发现问题时不要慌张：
- 按精准回退规则回退到对应环节
- 修改完成后重新提交审核
- **通过条件**：P0=0 且 P1≤2

### 6. 搜索结果正确性

- 搜索功能仅供选题调研和素材参考
- 搜索结果可能时效性不足或包含错误
- 涉及具体数据和引用的内容，建议手工核实后再发布

### 7. 文章保存路径

- 所有文章自动保存到 `F:\My_WenAn\OpenClaw-File\自媒体文章\`
- 以文章标题作为文件夹名
- 文件命名规则：`平台-标题首拼大写-文件类别-yyyymmdd-HHmm.md`
- 所有七虾环节产出直接归档到文章文件夹

## 八、常见问题解答

**Q1：七虾跑一半中断了怎么办？**

A：七虾有**自动中断恢复机制**。状态文件 `七虾运行时状态.json` 记录了当前进度和已完成阶段。恢复步骤：
1. 读取状态文件，确认当前阶段
2. 读取最后完成的交接文档
3. 从断点继续（从下一阶段开始）

**Q2：可以跳过某个环节吗？**

A：**不建议跳过**。每个环节的产出是下一环节的必需品。七虾是串联管线，跳过一个环节会导致后续环节缺乏必要信息。如果确实不需要（如不需要配图），在启动时说明即可。

**Q3：多平台同时发布怎么操作？**

A：文案虾会自动产出各平台版本（公众号版、小红书版、头条号版）。标题虾和图片虾也会为各平台分别生成标题和配图方案。一篇文章同时适配多平台。

**Q4：可以反复修改同一个环节吗？**

A：**可以**。手动模式下每环节都会等待确认，不满意可要求重做。审核虾也会在不通过时精准回退。全自动模式下如果需要修改，需要重新走一遍相关环节。

**Q5：全自动和手动模式怎么选？**

| 情况 | 推荐模式 |
|:----|:--------|
| 第一次使用 | 手动模式（每步确认，熟悉流程） |
| 赶时间 | 全自动模式 |
| 质量要求高 | 手动模式 |
| 批量生产 | 全自动模式 |
| 对方向不确定 | 手动模式 |

**Q6：七虾写作的内容会被判定为 AI 生成吗？**

A：七虾有多重去 AI 味机制：
- 文案虾专门做了去 AI 味润色
- 删除"首先/其次/最后"等AI套话
- 审核虾专门检查AI痕迹（维度⑧）
- 建议在产出后加入自己的个人风格和案例

**Q7：七虾支持什么语言？**

A：主要支持**中文内容创作**。如果需要在其他语言中使用，需要修改各虾的提示词模板。

**Q8：为什么我启动七虾没有反应？**

A：常见原因：
1. 技能包未正确安装 → 检查 `~/.openclaw/skills/` 下是否有8个技能文件夹
2. OpenClaw 未重启 → 安装技能后需要重启
3. 技能文件损坏 → 重新解压安装

## 九、进阶技巧

### 1. 选题矩阵法

**原理**：通过维度交叉生成无限选题，而不是等灵感。

**操作**：
```
主题 × 受众 × 内容形式 × 深度 = 选题矩阵
```

例如：
- 主题：AI办公 / 效率工具 / 副业赚钱
- 受众：新手 / 进阶 / 高手
- 形式：教程 / 测评 / 观点

3×3×3 = **27个选题**，填满一个月的内容日历。

更详细的选题矩阵搭建在 Obsidian 笔记「选题工厂与写作框架详解」中。

### 2. 写作框架组合技巧

不要只用单一框架，**组合使用效果翻倍**：

- **长视频/长文**：PPP 做整体结构 + 每个论点用 P&P 展开 + 开头用 PAS 做钩子
- **公众号文章**：PPP 为主体 + 文末用 PAS 做行动号召
- **小红书笔记**：PAS 抓开头 + 正文用 P&P 展开
- **知识星球内容**：PPP 深度长文 + 清单框架配附件

### 3. 跨平台分发策略

**一份素材，多平台适配**：
```
原始素材（深度干货）
    ├→ 头条号 → 拆成短篇连载（悬念标题）
    ├→ 公众号 → 完整深度文章（含搜索词标题）
    ├→ 小红书 → 精炼笔记（长尾词+精美封面）
    ├→ 知乎 → 回答问题（选对问题+专业深度）
    └→ 知识星球 → 独家深度（PPP框架+专属分析）
```

### 4. 关键词布局矩阵

**发布前检查清单**：
- [ ] 标题包含核心关键词（前置优先）
- [ ] 前100字自然嵌入关键词
- [ ] 小标题带长尾关键词
- [ ] 关键词密度 2%-8%
- [ ] 话题标签覆盖关键词（小红书）
- [ ] 文末互动引导

### 5. 发布后运营

**黄金1小时**（特别是小红书）：
- 自己先收藏
- 积极回复前几条评论
- 分享到私域/朋友圈获取初始流量

**48小时内**（特别是头条号）：
- 观察数据表现
- 回复评论区所有问题
- 不要频繁修改正文

### 6. 数据迭代闭环

```
发布 → 监测数据 → 发现问题 → 调整策略 → 再发布
                    ↑
              (持续循环)
```

**通用诊断**：

| 症状 | 可能原因 | 调整方向 |
|:----|:--------|:---------|
| 曝光/推荐低 | 标题/封面不够吸引 | 优化标题公式+封面图 |
| 阅读/点击低 | 开头不够钩人 | 前3秒/前50字重新打磨 |
| 完读率低 | 内容结构/段落太长 | 缩短段落+增加钩子密度 |
| 互动数据差 | 缺少引导 | 文末+评论区主动引导 |
| 搜索来源少 | 关键词布局不足 | 检查关键词匹配度+密度 |

### 7. 中断恢复技巧

如果七虾跑一半因为各种原因中断：

**查看状态文件**：
```json
// 七虾运行时状态.json
{
  "pipelineId": "auto-20260602-xxxx",
  "mode": "auto",
  "currentStage": 3,
  "completedStages": [1, 2, 3],
  "pendingStage": 4
}
```

**恢复操作**：
1. 读取状态文件确认中断位置
2. 读取 `七虾交接文档_阶段X.json` 获取上游信息
3. 从中断阶段+1 继续执行

### 8. CMO+SEO 优化要点

**CMO（内容营销优化）核心原则**：
- 头条号：悬念标题 + 钩子结构 + 日更
- 小红书：长尾关键词 + CES互动引导 + 黄金1小时
- 公众号：关键词精准布局 + 高质量内容 + 稳定更新

**SEO 核心原则**：
| 平台 | 优化重点 |
|:----|:---------|
| 公众号 | 公众号名称+标题+简介+正文关键词布局 |
| 小红书 | 标题+正文开头+话题标签+图片alt文字 |
| 知乎 | 选对问题 > 写好回答 > 初始互动运营 |
| 头条号 | 标题+标签+内容垂直度 |

### 9. 文章保存规范

**根目录**：`F:\My_WenAn\OpenClaw-File\自媒体文章\`

**文件夹规则**：以文章标题作为文件夹名

**文件命名规则**：
```
平台-标题前4字首拼大写-文件类别-yyyymmdd-HHmm.md
```

示例：
- `公众号-BZRZ-正文-20260605-2203.md`
- `小红书-BZRZ-标题-20260605-2203.md`

**平台标识**：公众号 / 头条号 / 小红书 / 知乎 / 知识星球 / 朋友圈

**文件类别**：标题 / 正文 / 图片 / 审核
